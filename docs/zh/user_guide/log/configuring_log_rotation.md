# 配置日志轮转

当文件大小超过每个日志文件大小和每个进程可写的日志文件个数乘积时，日志开始轮转。通过环境变量“MINDIE\_LOG\_ROTATE”设置日志的轮转。

每个日志文件大小的取值范围为1MB～500MB，默认为20MB；每个进程可写日志文件个数的取值范围为1个～64个，默认为10个。

设置某个组件的日志轮转格式为：_组件名称_: -fs _filesize_ -r _rotate_

- 如果“:”前无组件名称，则默认为对所有组件统一进行设置。
- 同时设置多个组件时用“;”隔开，且后方设置优先级高于前方设置，后方设置会覆盖前方设置。
- _filesize_表示每个日志文件大小，_rotate_表示每个进程可写日志文件个数。

    > [!NOTE]说明
    > 日志老化轮转的命名格式为：mindie-_组件名称_\__进程号_\__时间_._轮转文件编号_.log

【示例1】统一将MindIE所有组件日志文件大小设置为500MB。

```bash
export MINDIE_LOG_ROTATE="-fs 500"
```

【示例2】设置MindIE Torch日志文件大小不得超过40MB，每个进程可以写入一个文件。

```bash
export MINDIE_LOG_ROTATE="torch: -fs 40 -r 1"
```
